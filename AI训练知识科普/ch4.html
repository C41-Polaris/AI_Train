第四章·分布式训练策略

一张 GPU 不够？用十张、百张、千张。分布式训练的核心挑战和主流方案。

4.1 为什么需要分布式训练？

💡 三个动机

  1. 显存不够：70B 模型 FP16 权重就 140GB，一张 A100 80G 放不下
  2. 训练太慢：单卡训 70B 需要 200 年，一千张卡只需 70 天
  3. batch size 不够：大 batch 让梯度更新更稳定，单卡撑不起大 batch

4.2 Data Parallelism（数据并行）

最简单直接——把数据分到多张卡上，每张卡有完整的模型副本，处理不同数据。

Batch 0 [数据₁ 数据₂ 数据₃ 数据₄] ↓ ↓ ↓ ↓ GPU 0 GPU 1 GPU 2 GPU 3 [模型] [模型] [模型] [模型] ↓ ↓ ↓ ↓ All-Reduce ──────────────────────────────→ 同步梯度 ↓ 每张卡更新参数（更新后完全相同）

DDP（Distributed Data Parallel）

PyTorch 的标准数据并行实现：每张卡独立计算梯度，通过 Ring All-Reduce 高效同步梯度。

  • ✅ 实现简单——加一行代码即可
  • ✅ 通信效率高（Ring All-Reduce，通信量与卡数无关）
  • ❌ 每张卡存完整模型，显存占用大
from torch.nn.parallel import DistributedDataParallel
model = MyModel().to(device)
model = DDP(model, device_ids=[local_rank])
# 训练代码几乎不用改！

4.3 FSDP / ZeRO——省显存的数据并行

DDP 的问题是每张卡存了一个完整模型。FSDP（Fully Sharded Data Parallelism）和 DeepSpeed ZeRO 把模型参数、梯度、优化器状态分片到各卡上，用的时候再收集。

ZeRO 的三个 Stage

Stage 1: 分片优化器状态（显存省 ~4×） [模型: 全] [梯度: 全] [优化器状态: 分片] Stage 2: 分片梯度（显存省 ~6×） [模型: 全] [梯度: 分片] [优化器状态: 分片] Stage 3: 分片模型参数（显存省 ~N×） [模型: 分片] [梯度: 分片] [优化器状态: 分片] N = GPU 数量。4 卡训 7B FP16： DDP 需要: 56 GB（14×4） ZeRO-3 每卡: ~14 GB 模型参数（每次只加载 1/N） + 分片梯度/优化器 → 每卡约 20 GB
每卡显存（7B）每卡显存（70B）通信量
DDP~56 GB~560 GB（不可能）O(1)
ZeRO Stage 1~28 GB~280 GBO(1)
ZeRO Stage 2~12 GB~120 GBO(1)
ZeRO Stage 3~8 GB~20 GB（8卡分摊）O(N) 更多

⚡ ZeRO 实战口诀

显存够用就用 Stage 2（性能最优）
显存紧张上 Stage 3（但通信开销大）
单机多卡 NVLink：FSDP ≈ ZeRO Stage 3
多机多卡 Ethernet：优先 ZeRO Stage 2

4.4 模型并行（TP / PP）

当模型大到一张卡连一层都放不下时，需要把模型本身拆开。

Tensor Parallelism（张量并行）

把一个层的计算切分到多张卡上例如将 Attention 的 head 分给不同的 GPU。

一个 Transformer Layer ┌────────────────────────────────┐ │ Multi-Head Attention │ │ ┌───┬───┬───┬───┬───┬───┐ │ │ │ h₁│ h₂│ h₃│ h₄│ h₅│ h₆│ │ ← Head 分到 2 卡 │ └───┴───┴───┴───┴───┴───┘ │ │ ↓ all-reduce │ └────────────────────────────────┘ GPU1: heads 1-3 GPU2: heads 4-6 ┌──────────────────┐ ┌──────────────────┐ │ Q₁ K₁ V₁ → h₁-3 │ │ Q₂ K₂ V₂ → h₄-6 │ │ SwiGLU FFN 1/2 │ │ SwiGLU FFN 1/2 │ └──────────────────┘ └──────────────────┘ ↑ all-reduce after each matmul

Pipeline Parallelism（流水线并行）

把不同层放在不同 GPU 上，数据像流水线一样在 GPU 间传递。

GPU 0: Layer 1-10 ↓ (GPU 间通信) GPU 1: Layer 11-20 ↓ GPU 2: Layer 21-30 ↓ GPU 3: Layer 31-40

3D 并行 = Data Parallelism × Tensor Parallelism × Pipeline Parallelism，这是训练千亿参数模型的标准方案。

4.5 混合精度训练

用 FP16/BF16 计算梯度，用 FP32 维护参数。训练速度 ~2×，显存减半。

FP32: [1.8e-38 到 3.4e+38] 精度高，显存大 FP16: [6.1e-5 到 6.5e+4] 显存减半，范围窄 BF16: [1.2e-38 到 3.4e+38] 同 FP32 范围，精度低 TF32: FP32 尾数精度 + FP16 范围，NVIDIA Ampere 专用

推荐的混合精度策略

GPU推荐 dtype说明
A100 / H100BF16范围大，训练稳定，性能与 FP16 相当
V100 / T4FP16不支持 BF16，需配合 loss scaling
RTX 30/40 系列BF16 或 FP16BF16 稳定性更好

4.6 总结

📌 选择策略的决策指南

  1. 单卡能放下模型 → DDP（最简单高效）
  2. 卡多但卡小 → FSDP / ZeRO Stage 3
  3. 模型单卡放不下 → Tensor Parallelism + Pipeline Parallelism
  4. 千亿参数 → 3D 并行（DP + TP + PP）+ ZeRO
  5. 永远开混合精度 → BF16（首选）或 FP16 + loss scaling
