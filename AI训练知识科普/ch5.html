第五章·微调与高效训练

关键一章——教你在消费级 GPU 上训大模型的所有技巧。

5.1 Full Fine-Tuning vs LoRA

微调是把预训练好的模型在特定任务上做进一步训练。但不是一定要更新所有参数。

Full Fine-TuningLoRA
更新参数量100%（7B → 全部 7B 参数）~0.1-1%（~7M-70M）
显存需求（7B）~56 GB（FP16）~16 GB（FP16）
训练速度慢（全参数梯度计算）快 ~2-3x
效果上限高（完全适应新任务）接近 Full FT（低损失）

5.2 LoRA 详解

LoRA（Low-Rank Adaptation）：不直接修改原始权重 W，而是在 W 旁插入一个低秩分解的更新矩阵。

W_new = W_original + ΔW ┌────┐ ΔW ≈ A · B │ A │ (d×r) W_original │ │ r << d, 通常 r=8~64 ┌────────┐ │ │ │ W×x │ │ B │ (r×k) └────────┘ └────┘ + = W_new×x (A·B)·x

LoRA 的核心洞察：模型权重的"更新量"（ΔW）具有低秩特性，可以用两个小矩阵 A (d×r) 和 B (r×k) 来近似表示。

from peft import LoraConfig, get_peft_model

lora_config = LoraConfig(
    r=16,               # 秩——越大越强也越贵
    lora_alpha=32,      # 缩放系数
    target_modules=["q_proj", "v_proj"],  # 只微调 Attention 的 Q 和 V
    lora_dropout=0.05,  # Dropout 防过拟合
)
model = get_peft_model(base_model, lora_config)
# 现在 model 可训练参数只有原来的 ~0.1%！

🔑 关键超参数

r（rank）：r 越大，ΔW 表达能力越强，但参数量和显存也更大。
r=8（轻量，适合简单任务），r=16（均衡，推荐），r=64（强力，适合困难任务）
target_modules：哪些层加 LoRA。q_proj+v_proj（最常见，效果已很好），加 o_proj+gate_proj+up_proj+down_proj（更强但更耗显存）

5.3 QLoRA——量化 + LoRA

QLoRA = 4-bit 量化（NF4）+ LoRA。把基座模型量化到 4-bit，LoRA 部分保持 FP16。

显存分布对比（7B 模型，单卡训练）: 全量微调: ┌──────────────────────────────────────────┐ │ Model (FP16) 14G | Grad 14G | Opt 28G │ ≈ 56 GB ❌ └──────────────────────────────────────────┘ LoRA (FP16): ┌──────────────────────────────────────┐ │ Model (FP16) 14G | LoRA+Opt ~3G │ ≈ 17 GB ✅ └──────────────────────────────────────┘ QLoRA (4-bit NF4): ┌──────────────────────────────────────┐ │ Model (NF4) 4G | LoRA+Opt ~2G │ ≈ 6 GB 🎉 └──────────────────────────────────────┘
# QLoRA 在 LLaMA-Factory 中的配置
# dataset: my_data.json
# finetuning_type: lora
# quantization_bit: 4       ← 4-bit 量化！
# quantization_type: nf4    ← 4-bit NormalFloat

# 或者在 Transformers 中
model = AutoModelForCausalLM.from_pretrained(
    "meta-llama/Llama-3-8B",
    load_in_4bit=True,          # 4-bit 量化加载
    bnb_4bit_quant_type="nf4",  # NF4 数据类型
    bnb_4bit_use_double_quant=True,  # 双重量化再省 0.5GB
)

5.4 GaLore——梯度低秩投影

GaLore（Gradient Low-Rank Projection）是一种更激进的内存节省方法。它不是在权重上加 LoRA，而是在训练过程中将梯度投影到低秩空间，减少优化器状态的显存。

  • ✅ 可以 全量微调（只改变训练方式，不改模型结构）
  • ✅ 省优化器状态显存（Adam 优化器状态 ~28G → ~2G for 7B）
  • ✅ 最终模型 = 完整模型（不是 LoRA 那种需要合并的 adaptor）
  • ⚠️ 比 LoRA 慢一点（需要额外投影计算）

5.5 模型量化

量化是把模型从高精度（FP16）映射到低精度（INT8/INT4），减小模型体积和加速推理。

方法精度显存省适用场景
GPTQ4-bit / 3-bit / 2-bit~4xGPU 推理（需要校准集）
AWQ4-bit~4xGPU 推理（比 GPTQ 快）
GGUF (llama.cpp)Q2~Q8 多种~4-6xCPU/混合推理，消费级硬件
BitsAndBytes (NF4)4-bit NF4~4xQLoRA 训练 / 推理

5.6 训练优化技巧

Flash Attention

Flash Attention 是一种精确的 Attention 算法加速，不牺牲精度。

  • ✅ 训练速度 2-3x 加速
  • ✅ 显存从 O(n²) 降到 O(n)（长序列尤其明显）
  • ✅ 精确计算——结果和标准 Attention 完全一致

Gradient Checkpointing

训练时不保存中间激活值，反向传播时重新计算。

  • ✅ 显存省 ~50%（但慢了 ~20%）
  • ✅ 在显存不够时是救命稻草

5.7 实践建议

📌 实战清单

  1. QLoRA + r=16 + target_modules=q_proj,v_proj 是性价比最高的起点
  2. 数据量少（<500 条）用 r=8，数据量多（10000+）用 r=32+
  3. 无论如何开 Flash Attention 和 Gradient Checkpointing
  4. 7B 用 QLoRA 单张 24GB 卡足够，34B 用 QLoRA 需要 24GB
