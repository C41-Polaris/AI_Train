只 训练不管等于白 训练。如何科 学地判断模型 训练好了？

7.1 训练练时的生命线——Loss 曲线

💡 Loss 就是模型的"自评分数"

Loss 越低，模型预测越准。但 Loss 曲线要懂得看，不是越低就一定越好。

典型 Loss 曲线解读

Loss 曲线解读: 训练练 loss 持续下降 + 验证 loss 同步下降 → 学习正常 ✅ 验证 loss 下降更快 → 验证数据可能泄漏漏 ⚠️️ 验证 loss 开始上升 → 过拟合 ⚠️️ ⚠️️ 训练练 loss 剧烈震荡 → 学习率太大大 ⚠️️ 训练练 loss 不下降 → 学习率太大小 / 数据有 bug ❌ 关键指标: - Training Loss: 训练练集上的损失，反映拟合能力 - Validation Loss: 验证集上的损失，反映泛化能力（更关键！） - Perplexity (PPL): Loss 的指数形式
Perplexity = exp(Loss)
例如: Loss=2.3 → PPL≈1.0（模型平均在 1.0 个词中猜一个）
Loss=3.0 → PPL≈2.0
Loss=1.6 → PPL≈5 （GPT-4 级别）

7.2 过拟合与欠拟合

现象 训练练 Loss 验证 Loss原因怎么办
欠拟合高高模型容量不够/ 训练练不足加大模型、更多轮数、增大 学习率
过拟合低升高模型死记硬背 训练练集正则化、早停、更多数据、降低 LoRA rank
完美低接近 训练练 Loss模型 学到了真正规律✅

🧪 微调场景的特有问题

灾难性遗忘（Catastrophic Forgetting）：微调新任务后模型忘了原先的能力。
解法：混入 5-2.0% 的原预 训练练数据、EWC、低 学习率。

7.3 Benchmark 评测

MMLU

📝 57 个 学科的选择题考试

  • 5-shot，评分：准确率（0-1.00 分）
  • 及格线：人类专家 ~89.8%，GPT-4 ~86.4%，LLaMA-3-70B ~82%
  • 中文版：CMMLU、MMLU-Chinese

GSM8K

📝 8000 道小 学数 学应用题

  • 8-shot，chain-of-thought，评分：准确率
  • GPT-4 ~97%，LLaMA-3-70B ~93%

HumanEval

📝 164 道 Python 编程题

  • 0-shot，评分：pass@1（一次通 过率）
  • GPT-4 ~87%，DeepSeek-Coder-33B ~79%

7.4 人工评估——最终的裁判

  • 盲测 A/B Test：两个模型回答同问题，人评选哪个好
  • LMSys Chatbot Arena：ELO 评分，最可靠 LLM 排行榜
  • LLM-as-a-Judge：让 GPT-4 给模型回复打分

⚡ 最小可行评估流程

  1. 训练练中：盯着 validation loss 和 Perplexity
  2. 训练练后：跑 lm-evaluation-harness 做 MMLU + GSM8K
  3. 拿 3.0-50 个真实用例 做人工盲选对比
  4. 迭代：发现问题 → 分析 → 修数据 → 重 训练

7.6 总结

📌 核心要点

  1. 验证 Loss 比 训练练 Loss 重要——泛化能力才是真能力
  2. 过拟合是微调最常犯的错——用早停、Dropout、低 rank 控制
  3. Benchmark 是门槛——MMLU + GSM8K + HumanEval 是黄金三件套
  4. 人工评估不可替代——尤其是你的垂直领域
  5. 实 验记录是科 学方法——不从 W&B 看曲线找规律，就是随机碰运气
