第二章·Transformer 架构深度解析

没有 Transformer 就没有今天的 LLM。理解它的每一个组件的设计哲学。

2.1 从 RNN 到 Transformer——为什么需要 Attention？

在 Transformer 之前，序列建模的主角是 RNN（循环神经网络）。RNN 逐个 token 读取输入，维护一个隐藏状态 h_t 作为"记忆"。

💡 RNN 的致命伤

长距离依赖问题：一个句子开头的词对结尾的影响随着距离指数级衰减（梯度消失/爆炸）。当序列长度超过 100，RNN 基本上"记不住"开头说了什么。

RNN（逐词读取，串行，慢）: "我" → h₁ → "爱" → h₂ → "你" → h₃ → 输出 每一步依赖上一步，无法并行 ❌ Transformer（一次读完，并行，快）: ["我", "爱", "你"] → ┌──────────────────┐ │ Attention │ ← 每个词可以直接"看"所有词 └──────────────────┘ → 输出 所有词同时处理，并行 ✅ 每个词通过 Attention 直接关注句子中任意位置的词 ✅

2017 年 Google 的论文 "Attention Is All You Need" 彻底改变了 NLP。核心思想：用 Attention 机制替代 RNN 的循环结构。

2.2 Scaled Dot-Product Attention

Attention 的核心：给定一个 Query（查询），在一堆 Key-Value 对中找出最相关的信息。

Attention(Q, K, V) = softmax(Q · Kᵀ / √d_k) · V
一个简单的例子——翻译 "I love you" Query: "love"（当前要翻译的词） Key: ["I", "love", "you"]（源句子中每个词的身份） Value: ["我", "爱", "你"]（源句子中每个词的含义） 步骤: 1. Q·Kᵀ = [0.2, 1.0, 0.3] ← love 与每个词的相似度分数 2. 除以 √d_k 缩放（防止 softmax 梯度消失） 3. softmax → [0.1, 0.7, 0.2] ← 归一化到概率 4. × V → 0.1×"我" + 0.7×"爱" + 0.2×"你" = "爱"

🔑 除以 √d_k 的意义

当 d_k 很大时，点积 Q·K 的方差也大，softmax 的梯度会趋于 0。除以 √d_k 把方差稳定在 1 附近，梯度更好传播。

2.3 Multi-Head Attention

不是只做一次 Attention，而是做 h 次（通常是 8-32 次），每次用不同的线性投影把 Q/K/V 映射到不同的子空间。

MultiHead(Q,K,V) = Concat(head₁, ..., head_h) · Wᵒ
其中 head_i = Attention(Q·Wᵢ^Q, K·Wᵢ^K, V·Wᵢ^V)
[Q, K, V] ↓ ┌──────────────────────────┐ │ Split into h heads │ │ ┌────┐ ┌────┐ ┌────┐ │ │ │ h₁ │ │ h₂ │ .. │ h₈ │ │ ← 每个 head 在不同子空间做 Attention │ └────┘ └────┘ └────┘ │ └──────────────────────────┘ ↓ Concat ↓ Linear Projection (Wᵒ) ↓ Multi-Head Attention 输出

好处：每个 head 可以关注不同的关系——语法关系、语义关系、位置关系等，丰富了模型的表现力。

2.5 Positional Encoding

Attention 本身是排列不变的——"我打你"和"你打我"在 Attention 看来是一样的，因为词序信息没有被编码。

所以需要 Positional Encoding 告诉模型"每个词的位置"。

绝对位置编码（Sinusoidal）

PE(pos, 2i) = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))

用不同频率的正弦/余弦波来编码位置。

  • ✅ 可以外推到更长的序列（不需要训练）
  • ❌ 每个位置编码是固定的，无法反映相对位置关系

可学习位置编码（Learnable Positional Encoding）

直接把位置索引映射为可训练的嵌入向量。

  • ✅ 灵活，模型自己学会位置表示
  • ❌ 最大序列长度固定（比如 2048），超过就崩

2.6 RMS Norm（Root Mean Square Layer Normalization）

在 LLM 中，LayerNorm 被替换为 RMS Norm，它更简单、计算更快。

RMSNorm(x) = x / sqrt(mean(x²) + ε) · γ

相比 LayerNorm，RMSNorm 去掉了减均值的操作，只做"缩放"。

  • ✅ 计算量减少 ~5-10%（LLM 训练中很重要）
  • ✅ 效果与 LayerNorm 几乎一致
  • ✅ LLaMA、Qwen 等都用它

2.7 RoPE（Rotary Position Embedding）

RoPE 是目前 LLM 中最主流的相对位置编码方案。它不是把位置信息加到输入上，而是旋转 Q 和 K 向量。

RoPE: f(q, m) = q · cos(mθ) + q_rotated · sin(mθ)

💡 直觉

想象一个旋转的指针：位置 m 对应一个旋转角度，Q 和 K 旋转后做点积，结果天然蕴含了相对位置信息——越近的词点积越大，越远的词点积越小。

  • ✅ 天然编码相对位置（两个 token 之间距离的重要性）
  • ✅ 可以外推到训练时未见过的序列长度（LLaMA 3 从 8K 外推到 128K）
  • ✅ 不增加额外参数

2.8 其他关键组件

Feed-Forward Network (FFN) / MLP

FFN(x) = SwiGLU(x · W_gate) ⊙ (x · W_up) · W_down

每个 Transformer 层都有一个 FFN，对每个位置的表示做非线性变换。LLaMA 的 FFN 用的是 SwiGLU（有门控的 FFN）。

Residual Connection（残差连接）

output = Layer(x) + x

每个子层的输入直接加到输出上。这帮助梯度直接流回浅层，缓解梯度消失，让超深网络（100+ 层）变得可训练。

Pre-Norm vs Post-Norm

方案位置特点代表模型
Post-Norm（原始）LayerNorm 在子层之后原始 Transformer，训练不稳定原始 Transformer
Pre-Norm（现代标准）LayerNorm 在子层之前训练更稳定，warmup 步骤少GPT, LLaMA, Qwen

2.9 总结

📌 核心要点

  1. Attention 让每个 token 直接关注所有 token
  2. Multi-Head 从不同子空间学不同类型的注意力
  3. RoPE 实现相对位置编码且支持外推
  4. RMS Norm + Pre-Norm + Residual 让超深网络稳定训练
  5. SwiGLU FFN 提供更强的非线性表达能力
